How We Build Paths

Our whole autonomous system is built on Pedro Pathing and a command-based architecture. Here's how a routine goes from idea to match-ready.

Plan Path ‚Üí Pedro Visualizer ‚Üí On-Robot Test ‚Üí Tune + Adjust ‚Üí Match Ready

Pedro Pathing

All of our autonomous movement runs through Pedro Pathing. It handles the trajectory planning so we can focus on what the robot actually needs to do at each point on the field.

Command-Based

We use a command-based structure for our auto code. It makes it way easier to build routines out of reusable pieces and swap things around without rewriting everything.

Visualizer

Before anything touches the real robot, we run it through Pedro's visualizer first. It lets us check timing, path shape, and spacing without wasting field time.
